C++ Implementation

Local variables

Local variable info is provided by LocalVariableTable attribute in Code attribute for the method. The table contains starting PC and length of the valid range for each variable. Multiple variables can occupy same local variable table slot as long as their PC ranges do not overlap.

Starting PC for the variable has an issue: if variable is first assigned using xSTORE instruction, the starting PC for the variable is set to right after that xSTORE instruction. xSTORE instruction generation can't find that local variable because the instruction is just ahead of the range in which variable is available. To work around this issue, starting PC is rolled back by the length of the xSTORE instruction.
