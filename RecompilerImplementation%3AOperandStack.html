Java Recompiler: Instruction Implementation

Operand stack in Java bytecode is of limited size. The size is provided in the Java method Code attribute. The recompiler implementation uses unbound stack, with the assumption that valid Java code will not overflow the stack.

Each stack entry consists of type and name/expression, separated by '=' symbol. '=' is chosen because it is for now very unlikely to appear as a part of expression in the stack.

Type is necessary in many cases where type erasure in Java must be undone for C++. Stack entry is in same format as the locals entry that is being pushed to the stack.
