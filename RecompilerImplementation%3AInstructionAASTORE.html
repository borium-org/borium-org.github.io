Instruction AASTORE implementation

AASTORE instruction stores an expression into an array element.

AASTORE parameters in popping order:

  1. Value
  2. Index
  3. Array

At this time only AASTORE into a string array during initialization is implemented. Any other AASTORE instance may assert.

Array initializer follows this pattern:

  1. BIPUSH length
  2. ANEWARRAY String
  3. Zero or more of the following:
    1. DUP
    2. ICONST index, BIPUSH index, etc.
    3. LDC value, calculate value
    4. AASTORE
  4. ASTORE local

String array initializer is generated in a separately scoped block where temporary variable is used to reference the string array. All initializers use the temporary variable to assign their string value into an array element. At the end of the block, when ASTORE is being generated, the temporary variable is stored into the target of ASTORE. All this special handling is initiated when ANEWARRAY instruction allocates a string array.
