Java Recompiler: Instruction Implementation

NEW instruction creates an uninitialized object and pushes it to the operand stack.

NEW by itself does not do much. The instruction is always followed by DUP and INVOKESPECIAL. INVOKESPECIAL invokes the constructor for that new object, and even at that time the exact type is not known due to Java type erasure in generics. Only if the PUTFIELD is used to store the new object somewhere the type of the field will be used to complete the 'new' object type.
