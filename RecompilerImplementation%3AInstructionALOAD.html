Instruction ALOAD implementation

There are three variants of the ALOAD instruction:

  1. Constant embedded into instruction code: ALOAD 0...ALOAD 3
  2. Single byte constant: ALOAD n
  3. Double byte constant: WIDE ALOAD n

All these instructions perform the same task: Take a reference to any object and push it to the stack. The only difference is in the way how the index is specified. Few conditions must be satisfied:

  1. Index is within locals range.
  2. Local at that index must be defined. In non-static methods index 0 is 'this' reference. Some locals are defined through parameters that are passed into the method. Local variables are initially not defined, they get defined when ASTORE instructions are executed.

If these conditions are satisfied, the local variable is pushed to the operand stack.
