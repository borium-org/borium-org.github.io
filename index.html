Boring Web Site

This is a web site for my pet projects. The list is derived from Challenging projects every programmer should try and More challenging projects every programmer should try with some tweaks. Most people would probably find most of it just plain boring, but this stuff is interesting enough for me to try.

So what is boring?

  • Compiler parser generator: All compilers use something to parse the source code. Most parser generators generate state machine based parsers that are unfit for human consumption. The parser I'm toying with uses recursive descent, so the structure of the parser is much more similar to the structure of the grammar.
  • Small operating systems: In the past I developed a RTOS for 32-bit processors (ColdFire, ARM7, ARM9). It was lots of fun then, and would be a lot more fun now for 64-bit processors such as Intel x64 and ARM v8 now.
