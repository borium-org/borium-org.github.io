Instruction ASTORE implementation

There are three variants of the ASTORE instruction:

  1. Constant embedded into instruction code: ASTORE 0...ASTORE 3
  2. Single byte constant: ASTORE n
  3. Double byte constant: WIDE ASTORE n

All these instructions perform the same task: Pop a reference to any object from the operand stack and store it in a local variable. The only difference is in the way how the index is specified. Index must be within locals range.

Current implementation fails to store the value into a local variable slot that hasn't been used yet.
