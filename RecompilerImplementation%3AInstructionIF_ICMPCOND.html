Java Recompiler: Instruction Implementation

There are six variants of the IF_ICMPxx instructions, all of them compare two integers on top if stack with each other. If condition is satisfied, instruction at (address + offset) is executed next. Address is unsigned 16-bit location of the instruction in the bytecode array. Offset is a signed 16-bit value that is part of the instruction.

  1. Top of operand stack is popped as a right operand.
  2. Top of operand stack is popped as a left operand.
  3. Type of both top entries is verified to be int.
  4. if() statement is generated, with the body of statement generated as a goto statement to the target label. Top of stack expressions are surrounded with parentheses to avoid operator precedence issues.

Recompiler will not attempt to recognize structured programming constructs such as if(), while(), etc. This is out of scope of this program. Goto was considered harmful, but the code executes just the same if goto in the bytecode is translated into goto statement in C++ code.
