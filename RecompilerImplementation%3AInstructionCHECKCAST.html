Java Recompiler: Instruction Implementation

CHECKCAST is used to verify if top of the operand stack is of a specific type. In our case the check is unnecessary because we trust the generated bytecode. Instead, we update the type of the top of stack entry to the one that CHECKCAST verifies it to be.

One usage case, for example, is in the type-erased operation on a generic class: 

//		aload 2
//				stack[0]: ArrayList*=newClassNames
//		iconst 0
//				stack[0]: ArrayList*=newClassNames
//				stack[1]: int=0
//		invokevirtual java.util.ArrayList.remove
//				stack[0]: Object*=newClassNames->remove(0)
//		checkcast java.lang.String
//				stack[0]: String*=newClassNames->remove(0)
//		astore 3

INVOKEVIRTUAL operates on ArrayList and its return value is type-erased so it shows as Object*. CHECKCAST is inserted by the compiler to check if the result is convertible to String*. We assume that conversion is valid, so the stack entry type is changed to String* and last ASTORE 3 will succeed because last two bytecodes imply that local variable 3 is of type String*. When run-time type information is implemented, the CHECKCAST bytecode can also generate an actual verification if the stack entry type is valid. This will require much more information than what C++ RTTI provides, so it will be a separate effort some day.
