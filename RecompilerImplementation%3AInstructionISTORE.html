Java Recompiler: Instruction Implementation

There are three variants of the ISTORE instruction:

  1. Constant embedded into instruction code: ISTORE 0...ISTORE 3
  2. Single byte constant: ISTORE n
  3. Double byte constant: WIDE ISTORE n

All these instructions perform the same task: Pop an integer from the operand stack and store it in a local variable. The only difference is in the way how the index is specified. Index must be within locals range.

If a local variable slot hasn't been used yet, current implementation creates a new local C++ variable in the place where ISTORE instruction is used. This code will fail to compile if there are GOTO instructions that jump past the local variable initialization. This is the case with Java for() loops where code performs the initialization and the jumps to the far end of the loop to verify the condition.
