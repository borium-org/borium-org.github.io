Instruction INVOKESPECIAL implementation

INVOKESPECIAL is used to execute constructors.

All constructors are referenced by their type-erased class type, so for C++ code generation erased types must be restored using other means, the INVOKESPECIAL instruction does not provide enough information for restoring types.

Current implementation supports invoking the base class constructor without parameters from the constructor for this class, or invoking a parameterless constructor for a dynamically allocated object.

Parameters would be determined by parsing the constructor signature, but there are no code sequences where parameters are passed to the constructor, so parameter handling is postponed for now. Only parameterless constructors are supported at this time. Parameterized constructors are easy to add but all in due time.
