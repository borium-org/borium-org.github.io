Java Recompiler

Java Recompiler: Instruction Implementation

INVOKESPECIAL is used to execute constructors.

All constructors are referenced by their type-erased class type, so for C++ code generation erased types must be restored using other means, the INVOKESPECIAL instruction does not provide enough information for restoring types.

Parameters to the constructor are parsed using the constructor descriptor and they are popped from operand stack in right-to-left order.

Code generation is different for super constructor and new constructor. Super constructor invocation is injected into this class constructor definition header, before all references and their tracker objects are initialized. New object constructors are kept inline in the code where the object reference may be used or stored somewhere.
