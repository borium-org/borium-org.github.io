Java Recompiler: Instruction Implementation

GETSTATIC instruction gets the static field from some object and pushes it to the operand stack.

When static fields are accessed, Java performs the so-called class loading. During class loading,class static initializers, if any, are executed so that the static field is prepared to be accessed. C++ does not have similar functionality built in. Native C++ static initializers cannot be used because their execution order is non-deterministic.

Therefore, a separate global GetStatic() function is used to read static fields. The function first executes static initializer for the class from which the field is to be read, and then returns the field value. Class static initializer will have a static flag that prevents the initializer to run more than once. Static initializer would also be called from constructors, just in case if static initialization needs to be performed before the instance is initialized.
